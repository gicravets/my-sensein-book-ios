# AGENTS.md — my-sensein-book-ios (handoff for AI agents)

iOS app for **My.Sensein.Book** (dev: Kravitz Geroge). SwiftUI EPUB/FB2 reader with a
**dependency-free engine** (own zip/EPUB/FB2 parsers, paginated WKWebView reader).

## Branding
Product **My.Sensein.Book** · home-screen icon label **`book`** · bundle **`com.sensein.book`**
· scheme **`my-sensein-book`**. (Old name "MyReader"/`com.example.myreader` is retired.)

## ⚠️ Build rules / gotchas
- **Xcode is the exception to the Docker-only rule** — build the iOS app on the host.
- Project is generated by **XcodeGen**: edit `project.yml`, run `xcodegen generate`.
  New files under `Sources/` are auto-included. Custom Info.plist is generated from
  `project.yml` `info.properties` (camera usage + ATS `NSAllowsLocalNetworking` for http localhost).
  ```bash
  xcodegen generate
  xcodebuild -project my-sensein-book.xcodeproj -scheme my-sensein-book \
    -sdk iphonesimulator -destination 'generic/platform=iOS Simulator' build
  ```
- **Codable migration gotcha:** Swift's synthesized `Decodable` does NOT apply default
  values for missing non-optional keys → adding a `Book` field can wipe `library.json`.
  `Book` has a hand-written `init(from:)` (in an extension, keeps memberwise init) using
  `decodeIfPresent ?? default`. **Always add new persisted Book fields through that decoder.**
- Driving the simulator UI via computer-use can fail (screen capture / SCContentFilter);
  `xcrun simctl io <id> screenshot` works for viewing. There may be a separate
  "My.Sensein.MoneyFlow" app on the sim — that's a DIFFERENT user project, don't touch it.

## Layout
- `Sources/App` — `MySenseinBookApp` (@main), injects LibraryStore/ThemeManager/ServerConfig.
- `Sources/Core` — engine: MiniZip, EpubParser, Fb2Parser, BookParser, SearchIndex, LibraryStore.
- `Sources/Models/Book.swift` — `Book` (reflow-safe locators: position = chapterIndex+chapterFraction;
  highlight = chapter+text+occurrence; bookmark = chapter+fraction), Bookmark, Highlight, AppPaths.
- `Sources/UI` — LibraryView, ReaderScreen/Controller/WebView, CurlReaderView, AnnotationsView, etc.
- `Sources/Sync` — **server sync** (this is the active area):
  - `Keychain` (device key), `ServerConfig` (URL+key, isLinked),
  - `APIClient` (X-API-Key; claim pairing, getBooks, putProgression, get/create highlights+bookmarks),
  - `PairingView` (VisionKit QR scan + manual entry; "Синхронизировать" button),
  - `SyncService` (two-way reading-progress sync, match by title+author, newest-wins;
    push highlights/bookmarks with iOS locator encoded in `locator.value`, hex→named color).

## Sync model (important)
iOS uses reflow-safe locators; the server uses Readium locators — **not 1:1**. Synced now:
overall `progress` (0..1), `isFinished`, `lastReadAt` (two-way); annotations push-only.
**TODO:** annotation pull + precise-position locator bridge (iOS chapter+fraction ↔ Readium);
auto-sync; devices panel + key revoke. Live e2e (link via QR + sync) verify by hand/device.

## Pairing flow
Server web `/pair` shows a QR `{url,t}` → app scans (or manual URL+token) → `POST
/api/v1/auth/pair/claim` → device key in Keychain → `X-API-Key` on all calls. See
[backend AGENTS.md](https://github.com/gicravets/my-sensein-book-backend).
